metro-rio-xml-json wants to provide information about the Metro Rio system in a
machine readable format, on XML and on JSON.

Nowadays the XML schema is this:

<metro_rio>
	<stations>
		<station>
			<code>three-letter string</code>
			<name>string</name>
			<old_name>string (optional)</old_name>
			<start_date>YYYY-MM-DD string (optional)</start_date>
			<wheelchair_access>yes|no string</wheelchair_access>
			<bicycle_parking>yes|no string</bicycle_parking>
			<bicycle_parking_capacity>int (optional)</bicycle_parking_capacity>
			<entrances> (optional)
				<entrance>
					<entrance_name>string (optional)</entrance_name>
					<exit_only>yes|no string (optional)</exit_only>
					<working_hours>Mo-Sa XX:XX-XX:XX; Su XX:XX-XX:XX string (two letters for weekday and 24h time semicolon separated</working_hours>
				</entrance>
			</entrances>
		</station>
	</stations>
	<lines>
		<line>
			<code>string</code>
			<name>string</name>
			<sequence>string (dash separated)</sequence>
		</line>	
	</lines>
</metro_rio>

but the idea is to be very specific, including: ways, lines, accessibility
(elevators and escalators), stores, subway entrances etc.
Feel free to colaborate with the XML schema itself or by adding data by
forking the project or sending an email.

The JSON file is generated automatically from the XML by running it into JSON XSLT with the following:

cat metro-rio.xml | tr -d '\t' | tr -d '\n' | xsltproc JsonML.xslt > metro-rio.js
